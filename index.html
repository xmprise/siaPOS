매장에서 일어날 수 있는 작업을 고려하여 판매 시점 관리 프로그램이 제공하는 기능을 매장 내에서 일어나는 작업과 외부에서 일어나는 작업으로 나누어 각각 다른 환경에서 운용 할 수 있도록 하였다.

우선 매장 내부에서 쓰이는 POS 프로그램은 데이터의 수집 기능만을 담당한다. 상품의 주문 처리, 주문 내역 파악, 주문 계산이 매장 내에 쓰이는 POS 프로그램에 주요 기능인 동시에 매장 내에서 가장 중요한 작업 이므로 매출의 조회나 분석은 우선적인 주요 작업은 아니다. 따라서 이러한 작업은 WEB과 Mobile 환경으로 분리하여 매장 내부가 아닌 외부에서도 처리 가능하도록 하였다. 또한 매장 관리자는 장소에 제약을 받지 않고 POS에서 수집된 데이터를 관리 할 수 있기 때문에 보다 편한 관리가 가능하다는 장점이 있다.

다시 말하면 매장 내부의 작업자와 외부의 작업자를 나누어 이들이 다루기 편리한 사용환경에 맞는 프로그램을 제공 한다는 것이다. 이러한 작업이 가능 하도록 매장 내부에서 쓰이는 Windows Application과 외부에서 쓰이는 WEB Application, Mobile Application은 서로 통합된 데이터베이스를 사용하도록 하여, 매장에서 일어나는 매출 상황을 실시간으로 파악 할 수 있도록 구성 하였다.
